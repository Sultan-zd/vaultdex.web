⚠️
Un audit manuel prend plusieurs jours. Les outils existants sont fragmentés : certains font de l'analyse statique, d'autres de l'extraction ADB, aucun ne fait tout automatiquement. Il n'existe pas de solution bout-en-bout qui prend un APK et sort un rapport complet — jusqu'à VaultDex.
Notre Solution

Une plateforme d'analyse dynamique complète

Pas un simple scanner statique — VaultDex exécute réellement l'application sur un émulateur pour capturer son empreinte de stockage réelle.

Analyse Dynamique

L'APK est installé sur un émulateur Genymotion réel. Le comportement de l'app est simulé pour capturer les données écrites au runtime.

28 Règles de Sécurité

Credentials, PII, cryptographie faible, configurations dangereuses — 4 catégories, 28 patterns de détection basés sur des regex ciblées.

Score de Risque 0–100

Algorithme déterministe : Critical = 25 pts, High = 10 pts, Medium = 3 pts, Low = 1 pt. Répartition claire par niveau de criticité.

Export PDF Natif

Rapport complet généré côté client via jsPDF — en un clic, sans serveur supplémentaire, prêt à être remis à l'équipe de développement.

Plateforme Multi-Tenant

Authentification JWT stricte — chaque auditeur ne voit que ses propres scans. Rate limiting 5 req/min, headers sécurité, validation magic bytes.

Temps Réel via SSE

Chaque étape du pipeline est streamée vers le navigateur via Server-Sent Events — sans polling, l'utilisateur voit l'avancement en direct.

Moteur d'Analyse

28 règles, 4 catégories de vulnérabilités

Chaque règle est un pattern regex ciblé appliqué à tous les fichiers extraits — SharedPreferences, SQLite, fichiers génériques et cache applicatif.

Credentials
8 règles
  • Mots de passe en clair
  • Tokens JWT complets
  • Clés API cloud (AWS, Firebase, Stripe)
  • Identifiants OAuth / secrets Twilio
Données Personnelles (PII)
7 règles
  • Adresses e-mail
  • Numéros de téléphone
  • Numéros de carte bancaire
  • Numéros de sécurité sociale
  • Adresses IP internes
Cryptographie Faible
6 règles
  • Clés AES/RSA exposées en clair
  • Algorithmes obsolètes (MD5, SHA1)
  • Seeds d'entropy insuffisants
  • IV / nonce réutilisés
Configurations Dangereuses
7 règles
  • debuggable=true en production
  • allowBackup=true
  • Composants exportés sans permission
  • Flags réseau dangereux
